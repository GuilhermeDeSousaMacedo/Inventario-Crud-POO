# Inventário-Crud-POO
## *Trabalho avaliativo referente às sprints 3,4 e 5 do Curso de Cibersegurança da faculdade Federal de Uberlândia.

De acordo com o meu entendimento acerca do projeto apresentado criei mais uma vez um Sistema de Inventário Crud(como o do último trabalho), porém utilizando dos novos conhecimentos adquiridos por meio das novas sprints em que dentre eles incluim às seguintes bases do projeto:
### -Programação orientada à objeto em Python.
### -Utilização do docker por 24h como forma de dar veracidade ao projeto e sua funcionalidade no mundo moderno de hoje(resolução do dilema:na minha máquina funciona).
### -Utilização de Sql como banco de dados no formato Json.
## •Funcionalidades:
### •Cadastro de equipamentos de TI.
### •Cadastro de vulnerabilidades associadas a cada equipamento.
### •Consulta de equipamentos por tipo e ID.
### •Atualização de informações dos equipamentos.
### •Exclusão de equipamentos cadastrados.
### •Atualização de vulnerabilidades existentes.
### •Armazenamento permanente dos dados em arquivo JSON.
### •Carregamento automático dos dados ao iniciar o programa.
### •Interface interativa via terminal (CLI).
### •Geração automática de IDs únicos para equipamentos e vulnerabilidades.
## •Tipos de equipamentos suportados
### •💻 Laptop
### •🖥️ Servidor
### •🌐 Roteador
### •🖨️ Impressora
## •Informações armazenadas para cada equipamento
### •ID
### •Tipo
### •Hostname
### •Responsável
### •Setor
### •Lista de vulnerabilidades
### •Informações armazenadas para cada vulnerabilidade
### •VulnID
### •Descrição
### •Categoria
### •Severidade
### •Status
